# 设计文档 v3.2，第 10.4 节
# =============================================================================

## 1. 创建应用

1. 登录 [飞书开放平台](https://open.feishu.cn/)
2. 开发者后台 → 创建企业自建应用
3. 应用名称：`活动内容生产机器人`
4. 获取 `App ID` 和 `App Secret` → 填入 `deploy/.env`

## 2. 权限配置

在「权限管理」中添加以下 scope：

### 多维表格
- `bitable:app` — 多维表格读写

### 消息与群聊
- `im:message` — 发送消息
- `im:message:send_as_bot` — 以机器人身份发送消息

### 文件（如需上传产物到飞书）
- `drive:drive` — 访问云空间

**注意**：权限添加后需发布应用并管理员审批。

## 3. 多维表格自动化触发（推荐方案 A）

### 方案 A：多维表格自动化（推荐，最简单）

1. 打开飞书多维表格 → 右上角「自动化」
2. 新建自动化 → 触发器：**「当记录被创建」**
3. 动作：**「发送 HTTP 请求」**
4. 配置：
   - 方法：POST
   - URL：`https://your-domain.com/webhook/wf0-dispatcher`
   - Headers：`X-Internal-Token: <INTERNAL_WEBHOOK_TOKEN>`
   - Body（JSON）：
     ```json
     {
       "record_id": "{{record_id}}",
       "fields": {
         "活动名称": "{{活动名称}}",
         "活动时间": "{{活动时间}}",
         "一句话简介": "{{一句话简介}}",
         "活动详情": "{{活动详情}}",
         "视觉风格": "{{视觉风格}}",
         "海报模板": "{{海报模板}}",
         "推文类型": "{{推文类型}}",
         "生成视频": "{{生成视频}}",
         "视频类型": "{{视频类型}}",
         "视频时长": "{{视频时长}}",
         "视频旁白风格": "{{视频旁白风格}}",
         "涉及地图": "{{涉及地图}}",
         "特殊要求": "{{特殊要求}}",
         "再生成意见": "{{再生成意见}}"
       }
     }
     ```
5. 保存并开启

**优势**：无需处理事件加密/URL challenge/Ticket 验证，配置简单可靠。

### 方案 B：事件订阅（备用，需要完整 URL 验证链路）

1. 在「事件订阅」中订阅 `bitable.record.created_v1` 事件
2. 配置请求地址 URL：`https://your-domain.com/webhook/wf0-dispatcher`
3. 飞书会先发送 URL challenge → n8n 的 Webhook 节点需返回 challenge token
4. 此后所有事件回调需要解密 Encrypt Key
5. 传入的 HTTP body 需校验 Verification Token

**此方案较复杂，仅当方案 A 无法满足需求时使用。**

## 4. Token 刷新

飞书 tenant_access_token 有效期 2 小时。

在 n8n 中实现：

```
[飞书凭证节点] → 用 app_id + app_secret 换 token
   → 缓存 token + expires_at
   → 每次调用前检查：如果 expires_at - now < 5min → 刷新
```

n8n 的飞书节点原生支持 token 管理，无需手写逻辑。

## 5. 消息卡片配置

审核人的「通过/驳回/重新生成」按钮通过飞书消息卡片实现。

卡片 JSON 模板见 `feishu/card_template.json`。

回调：卡片按钮点击 → POST 飞书回调到 n8n 审核 webhook。

## 6. 机器人配置

- 机器人名称：活动生产助手
- 机器人描述：AI 内容生产流程通知
- 消息接收：开启
